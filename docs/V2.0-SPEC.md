# Lumen v2.0 Requirements & Design Spec

## Overview
Lumen v2.0 focuses on filling the UX gaps that prevent it from being a daily-driver PDF reader for researchers and power users. This spec covers 14 features across bug fixes, P0 (must-have), P1 (high-value), and P2 (polish) tiers.

---

## Bug Fixes

### BF-1: Search Debounce + Annotation List Reactivity
**Problem:** Search fires on every keystroke without debounce. Annotation list caches filtered results in `@State`, missing live updates.
**Design:**
- Add 300ms debounce to `SearchViewModel.performSearch`
- Replace `@State items` in `AnnotationListView` with computed property from `sidebarVM`

### BF-2: Recent Files Cleanup + Drop Visual Feedback
**Problem:** No way to remove stale entries from recent files. No visual feedback when dragging PDF onto the window.
**Design:**
- Add context menu (right-click) on recent file rows with "Remove from List" and "Clear All"
- Add `@State isTargeted` binding to `.onDrop` for blue-border highlight

## P0 Features

### P0-1: Keyboard Page Navigation
**Requirement:** Arrow keys (Up/Down or Left/Right), Space, Page Up/Down for page scrolling/turning.
**Design:** Add `.onKeyPress` handlers in `PDFReaderView` for arrow-down/space to scroll forward, arrow-up to scroll backward. Delegate to `PDFView`'s native scroll behavior where possible.

### P0-2: Page History Navigation (Back/Forward)
**Requirement:** After jumping (via bookmark, outline, search), user can return to previous position.
**Design:** Add `NavigationHistory` to `ReaderViewModel` — a stack of `(pageIndex, scrollOffset)` entries. Push on every jump, pop on "back". Toolbar gets back/forward buttons. Keyboard: `Cmd+[` / `Cmd+]`.

### P0-3: Auto-Reopen Last Document
**Requirement:** On launch, restore the last open document and reading position instead of showing the recent files screen.
**Design:** Store `lastOpenedFilePath` in `WindowStateManager`. On app launch in `LumenApp`, if a valid path exists, trigger `docVM.open(url:)`. User can opt out via a preference later.

## P1 Features

### P1-1: Dark / Eye-Care Reading Mode
**Requirement:** Allow users to set a dark or warm-tinted background behind PDF pages for comfortable reading.
**Design:** Add `ReadingMode` enum (`.normal`, `.dark`, `.sepia`, `.eyeCare`) to `ReaderViewModel`. Apply via `PDFView`'s `backgroundColor` and a CIFilter-based color inversion. Toolbar toggle button.

### P1-2: Two-Page Spread View
**Requirement:** Display two pages side by side (like an open book).
**Design:** Add `DisplayMode` enum (`.singleContinuous`, `.single`, `.twoContinuous`, `.two`) to `ReaderViewModel`. Map to `PDFView.displayMode` (.singlePage, .singlePageContinuous, .twoUp, .twoUpContinuous). Toolbar toggle.

### P1-3: Text Selection Floating Toolbar
**Requirement:** When user selects text, show a small floating toolbar with Copy / Highlight / Underline / Search actions.
**Design:** New `SelectionToolbarView` positioned near the selection rect. Detected via `PDFViewSelectionChanged` notification (already observed). Position calculated from selection bounds.

### P1-4: Multi-Window Document Support
**Requirement:** Open multiple PDFs in separate windows.
**Design:** Move `DocumentViewModel` creation into `MainWindowView` (per-window) instead of sharing one instance from `App`. Each window gets its own state. `WindowGroup` already supports this in SwiftUI.

## P2 Features

### P2-1: Reading Progress Bar
**Requirement:** Show a thin progress bar at the bottom of the reader indicating how far through the document the user is.
**Design:** Horizontal `ProgressBar` below the PDF view area, driven by `currentPage / totalPages`.

### P2-2: Status Bar
**Requirement:** Bottom bar showing current page, total pages, zoom level, and file size.
**Design:** New `StatusBarView` component at the bottom of the reader layout. Uses monospaced digits.

### P2-3: Document Properties Viewer
**Requirement:** Show PDF metadata (title, author, subject, creator, creation date, modification date, file size, page count, encryption status).
**Design:** New `DocumentInfoView` as a sheet/popover. Accessible via toolbar button or menu.

### P2-4: Distraction-Free Reading Mode
**Requirement:** Hide toolbar, sidebar, and title bar; show only the PDF content. Toggle with `Cmd+\` or a toolbar button.
**Design:** New `isDistractionFree` state. When active, hide all chrome via conditional rendering. Use `.toolbar(.hidden, for: .windowToolbar)` in SwiftUI.

### P2-5: Annotation Undo/Redo
**Requirement:** Support Cmd+Z / Cmd+Shift+Z for all annotation operations, not just drawing.
**Design:** Wrap annotation create/delete/update operations with `UndoManager` integration. Register undo actions in `AnnotationViewModel`.

### P2-6: Document Type Registration
**Requirement:** Register Lumen as a handler for `.pdf` files in Finder.
**Design:** Add `CFBundleDocumentTypes` to Info.plist via the Xcode project settings or a generated Info.plist.
