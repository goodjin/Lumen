# 产品需求文档 (PRD) - PDF阅读器 (macOS)

## 文档信息

- **项目名称**: PDF-Ve（PDF Viewer Enhanced）
- **版本**: v1.0
- **创建日期**: 2026-04-01
- **对标物**: Skim（核心阅读体验）+ PDF Expert（标注工具）
- **平台**: macOS
- **状态**: 草稿

---

## 1. 项目概述

### 1.1 背景与目标

**背景**: 现有macOS PDF阅读器各有取舍——Preview太简陋、Adobe太臃肿、PDF Expert太贵、Skim界面陈旧。作为个人使用的工具，需要一个快速、干净、专注于阅读和标注体验的PDF阅读器。

**目标**: 开发一款个人使用的macOS原生PDF阅读器，提供流畅的阅读体验和完善的标注工具，性能对标Skim，标注体验对标PDF Expert。

**对标参考**:
- 核心架构参考 **Skim**：轻量快速、标注数据独立存储、标注侧栏设计
- 标注工具参考 **PDF Expert**：工具种类全、交互直观
- 减量：去掉PDF编辑（文本/图片修改）、表单填写、签名、OCR等
- 减量：去掉云同步、企业级功能、协作功能

### 1.2 目标用户

| 用户类型 | 描述 | 核心需求 |
|---------|------|---------|
| 个人用户（作者本人） | 日常阅读PDF文档、技术书籍、论文 | 快速打开、流畅阅读、标注管理 |

### 1.3 核心价值主张

一款极致轻量的macOS PDF阅读器，专注于阅读和标注，启动即用，性能优先，标注完善。

### 2.2 用户故事

#### US-001: 打开PDF文件

**格式**: 作为用户，我想快速打开一个PDF文件，以便开始阅读。

**详细描述**:
- **角色**: 用户
- **功能**: 打开PDF文件
- **价值**: 快速进入阅读状态
- **对标参考**: Skim和Preview都支持拖拽、双击、菜单打开

**验收标准**:
- AC-001-01: 用户可以通过菜单「文件 > 打开」选择PDF文件打开
- AC-001-02: 用户可以将PDF文件拖拽到应用窗口或Dock图标上打开
- AC-001-03: 用户可以通过系统文件关联双击PDF文件打开（可选为默认PDF阅读器）
- AC-001-04: 当打开一个新文件时，应用在3秒内显示第一页内容（针对100MB以内的文件）
- AC-001-05: 当文件不是有效PDF时，应用显示错误提示信息

**边界条件**:
- 输入: 任意.pdf文件，文件大小不超过500MB
- 输出: PDF内容渲染显示在主视图区域
- 异常: 文件损坏、文件不存在时显示错误对话框

#### US-002: 浏览PDF页面

**格式**: 作为用户，我想流畅地浏览PDF各个页面，以便阅读文档内容。

**验收标准**:
- AC-002-01: 用户可以通过鼠标滚轮或触控板手势上下滚动翻页
- AC-002-02: 用户可以使用键盘方向键或Page Up/Page Down翻页
- AC-002-03: 当前页码在工具栏中显示，格式为「当前页 / 总页数」
- AC-002-04: 用户可以在页码输入框中输入数字并按Enter跳转到指定页
- AC-002-05: 滚动时页面渲染流畅，不出现明显卡顿（目标60fps）

**边界条件**:
- 输入: 滚动手势、键盘按键、页码输入
- 输出: 对应页面内容显示在视图中
- 异常: 输入页码超出范围时，跳转到最近的有效页（第1页或最后一页）

#### US-003: 缩放页面

**格式**: 作为用户，我想调整PDF的显示比例，以便在不同情况下舒适阅读。

**验收标准**:
- AC-003-01: 用户可以使用Cmd+加号/减号缩放，每次步进10%
- AC-003-02: 用户可以使用Cmd+0还原到100%显示
- AC-003-03: 用户可以使用Cmd+1切换到「适合宽度」模式（页面宽度铺满视图）
- AC-003-04: 用户可以使用Cmd+2切换到「适合页面」模式（整页显示在视图内）
- AC-003-05: 用户可以使用触控板双指捏合手势缩放
- AC-003-06: 缩放范围限制在10%到500%之间
- AC-003-07: 当前缩放比例显示在工具栏中

**边界条件**:
- 输入: 键盘快捷键、触控板手势
- 输出: 页面按指定比例显示
- 异常: 缩放到边界值时不再响应该方向的缩放操作

#### US-004: 使用目录导航

**格式**: 作为用户，我想查看和使用PDF的目录，以便快速跳转到指定章节。

**验收标准**:
- AC-004-01: 当PDF包含目录时，侧栏「目录」标签中显示层级目录树
- AC-004-02: 用户点击目录项时，主视图跳转到对应页面
- AC-004-03: 目录支持多级层级缩进展示（最多支持5级）
- AC-004-04: 用户可以折叠/展开目录层级
- AC-004-05: 当PDF不包含目录时，目录标签显示「此文档无目录」提示
- AC-004-06: 用户可以通过快捷键Cmd+T显示/隐藏侧栏

**边界条件**:
- 输入: 目录项点击
- 输出: 主视图跳转到对应页面
- 异常: 目录项指向的页码超出范围时，跳转到最后一页

#### US-005: 全文搜索

**格式**: 作为用户，我想在PDF中搜索关键词，以便快速找到相关内容。

**验收标准**:
- AC-005-01: 用户可以通过Cmd+F打开搜索栏
- AC-005-02: 用户输入关键词后，所有匹配项在PDF中高亮显示（黄色背景）
- AC-005-03: 搜索结果数量显示在搜索栏中（如「3/12」）
- AC-005-04: 用户可以通过Enter或Cmd+G跳转到下一个匹配项
- AC-005-05: 用户可以通过Shift+Enter或Cmd+Shift+G跳转到上一个匹配项
- AC-005-06: 搜索不区分大小写（默认），可选「区分大小写」
- AC-005-07: 当没有匹配结果时，搜索框背景变红，显示「未找到」
- AC-005-08: 用户按Esc关闭搜索栏，搜索高亮消失

**边界条件**:
- 输入: 文本关键词（1-100个字符）
- 输出: 匹配内容高亮，视图跳转到第一个匹配位置
- 异常: PDF中无可搜索文本（扫描件）时提示「此文档不包含可搜索文本」

#### US-006: 高亮标注文本

**格式**: 作为用户，我想对PDF中的文字进行高亮标注，以便标记重要内容。

**验收标准**:
- AC-006-01: 用户选中文本后，工具栏显示标注工具按钮，或右键菜单显示「高亮」选项
- AC-006-02: 用户可以选择高亮颜色（至少支持：黄色、绿色、蓝色、粉色、橙色）
- AC-006-03: 高亮标注后，该文本区域呈现对应颜色的半透明背景
- AC-006-04: 高亮标注数据持久化保存（重新打开文件后标注仍然存在）
- AC-006-05: 用户可以右键高亮标注，选择「删除标注」来移除
- AC-006-06: 用户可以在标注上双击添加文字注释

**边界条件**:
- 输入: 鼠标选中文本区域
- 输出: 选中文本显示高亮颜色
- 异常: 无法选择文本的区域（图片、扫描件）不触发高亮操作

#### US-007: 下划线标注

**格式**: 作为用户，我想对文本添加下划线标注，以便标记次要内容。

**验收标准**:
- AC-007-01: 用户选中文本后，可以选择「下划线」标注类型

**边界条件**:
- 输入: 鼠标选中文本区域

- 异常: 同高亮标注

#### US-008: 删除线标注

#### US-009: 添加文字注释

#### US-010: 自由绘制

### 2.3 数据实体

#### Entity-001: 文档（Document）

| 字段名 | 类型 | 约束 | 描述 |
|-------|------|------|------|
| filePath | String | 非空 | PDF文件绝对路径 |
| fileName | String | 非空 | 文件名（不含路径） |
| pageCount | Int | 非空, > 0 | 总页数 |
| lastOpenedAt | Date | 非空 | 最后打开时间 |
| lastViewedPage | Int | 默认1 | 最后阅读页码 |
| zoomLevel | Float | 默认1.0 | 上次关闭时的缩放比例 |

**实体关系**:
- 与 Annotation: 一对多（一个文档有多个标注）
- 与 Bookmark: 一对多（一个文档有多个书签）

#### Entity-002: 标注（Annotation）

| 字段名 | 类型 | 约束 | 描述 |
|-------|------|------|------|
| id | UUID | PK | 唯一标识 |
| documentPath | String | FK | 所属文档路径 |
| type | Enum | 非空 | 类型：highlight/underline/strikethrough/note/drawing |
| pageNumber | Int | 非空, > 0 | 所在页码 |
| color | String | 非空 | 颜色（hex格式） |
| content | String | 可空 | 文字注释内容 |
| selectedText | String | 可空 | 被标注的原文文字（高亮/下划线/删除线） |
| bounds | Rect | 非空 | 标注在页面上的位置和大小 |
| drawingData | Data | 可空 | 绘制路径数据（仅drawing类型） |
| createdAt | Date | 非空 | 创建时间 |

#### Entity-003: 书签（Bookmark）

| 字段名 | 类型 | 约束 | 描述 |
|-------|------|------|------|
| id | UUID | PK | 唯一标识 |
| documentPath | String | FK | 所属文档路径 |
| pageNumber | Int | 非空, > 0 | 书签页码 |
| name | String | 非空 | 书签名称（默认「第X页」） |
| createdAt | Date | 非空 | 创建时间 |

### 2.4 业务流程

#### Flow-001: 标注创建流程

**流程描述**: 用户对文本进行选择并创建标注的流程

**状态定义**:
| 状态 | 描述 |
|-----|------|
| 空闲 | 未选中任何文本，工具栏标注按钮不可用 |
| 文本已选中 | 用户选中了文本，工具栏标注按钮变为可用 |
| 标注已创建 | 标注创建完成，保存到本地存储 |

**状态转换**:
| 当前状态 | 触发事件 | 下一状态 | 条件 |
|---------|---------|---------|------|
| 空闲 | 用户选中文本 | 文本已选中 | 选中区域包含可识别的文本 |
| 文本已选中 | 用户点击标注类型 | 标注已创建 | 无 |
| 文本已选中 | 用户点击空白区域 | 空闲 | 取消选中 |
| 标注已创建 | 无 | 空闲 | 标注创建后自动回到空闲状态 |

#### Flow-002: 标注持久化流程

**流程描述**: 标注数据的保存和加载流程

**对标参考**: Skim将标注保存在独立的元数据文件或PDF的扩展属性中，不修改原始PDF

**状态定义**:
| 状态 | 描述 |
|-----|------|
| 文档已打开 | PDF文件已加载，尝试读取已有标注 |
| 标注已加载 | 历史标注从本地存储读取并渲染到页面 |
| 标注已保存 | 新增/修改/删除的标注写入本地存储 |

**规则**:
- 当创建、修改、删除标注时，立即触发保存操作（非延迟）
- 标注数据存储在应用的专属目录（不修改原始PDF文件）
- 标注通过文档的绝对路径关联到对应文档

### 4.3 最佳实践建议

**推荐采用**:
1. **标注侧栏设计**: Skim的标注列表侧栏被学术用户高度好评，清晰展示所有标注并支持快速跳转，必须实现
2. **标注数据独立存储**: Skim的不修改原始PDF策略更安全、更可靠，避免数据损坏风险
3. **多色高亮**: PDF Expert提供5种高亮颜色，满足分类标注需求，成本低价值高
4. **记忆阅读位置**: 所有竞品都支持，属于基础体验，必须实现

**建议避免**:
1. **功能膨胀**: Adobe Acrobat因功能过多被批评为「核电站控制面板」，个人工具应保持简洁
2. **云同步复杂性**: 引入云同步会带来网络依赖、冲突解决等复杂度，个人工具本地存储即可
3. **PDF文本编辑**: 属于高复杂度低频功能，跳过可大幅降低开发复杂度

**个人工具优化机会**:
1. **极简UI**: 不需要企业级功能的入口和设置，界面可以比所有竞品更干净
2. **即开即用**: 不需要账号、登录、许可证验证，直接打开即用
3. **macOS原生**: 使用SwiftUI/AppKit原生技术栈，在Apple Silicon上性能最优
